## Decisions

- `--approval` maps to Codex config via `-c approval_policy=...` because local `codex exec` does not expose a direct approval flag.
- AgentQ config `context_file` is mapped to Codex project instruction discovery through `project_doc_fallback_filenames`; it is not injected into the prompt body.
- Phase 1 stores `run.json`, `stdout.jsonl`, `stderr.log`, and `output.md` for each run.
- AgentQ-owned files use `.agentq` and `~/.agentq`; the earlier generic planning path was corrected.
- `bun run check` now includes `gts lint`; `bun run fix` delegates to `gts fix`.
- Agent Markdown frontmatter is parsed by `front-matter`; AgentQ only validates the resulting attributes and required anchors.
- Agent runtime frontmatter must explicitly set `provider`, `model`, and `reasoning`; use `reasoning: none` when the selected model has no reasoning effort.
- Agent frontmatter requires `description` so humans and harness reports can identify the agent's job without opening the file.
- `fun` lives at `.agentq/agents/fun.md` and is intentionally read-only; this project sets `README.md` as the context file in `.agentq/config.json`.
- Codex provider always passes `--skip-git-repo-check` so AgentQ can run from non-git project directories.
- Run directories keep source agent/context paths in `run.json`; they no longer copy the agent and context files.
- The rendered `<artifacts>` block includes the run's `artifacts/` path; agents should only create files there when the task or artifact instructions call for it.
- Canonical run syntax is `agentq run <agent> --task <task>`.
- CLI run summaries default to a compact human-facing view with agent id, status, duration, run directory, tool/edit counts, changed files, failure details, and final `output.md`; `--details` prints the full card with artifact paths and metadata.
- `--verbose` remains the live event timeline mode and now also prints the detailed final summary.
- The live spinner uses `AgentQ <agent> running`.
- Context-file precedence is `--context-file`, then `.agentq/config.json`, then `~/.agentq/config.json`, then Codex defaults; agents do not define context files.
- Phase 2 keeps raw Codex output in `stdout.jsonl` and derives stable AgentQ event summaries for metadata/progress.
- Pretty terminal output is rendered from normalized events with Chalk; JSONL and metadata remain the machine-readable source of truth.
- Timeout status wins over provider exit code, even when the provider reports exit code `0`.
- Run history is read from existing `run.json` files under `~/.agentq/runs`, sorted by `startedAt` newest first, with optional `--since` durations (`1h`, `7d`, `2w`) and `--limit`.
- The reusable AgentQ skill intentionally lives at `skill/` instead of `.agents/skills` so it can be installed elsewhere; its frontmatter skill name is `agentq`.
- The skill's reliability guidance distinguishes final `output.md`, run `artifacts/`, and harness-owned logs, and gives explicit stop conditions plus prompt patterns for dependable agents.
- Plain structured text is recommended for human-facing agents; JSON-only final output is recommended when a harness or orchestrator must parse the result and provide retry feedback.
