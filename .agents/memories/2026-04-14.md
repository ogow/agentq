## Decisions

- Authored agents must now choose `result_mode: plain` or `result_mode: json` in YAML frontmatter.
- `plain` is for human-readable structured final output.
- `json` is for harnesses, orchestrators, scripts, or later agents that need to parse the final result.
- AgentQ validates result mode and records it in run metadata, but it does not yet parse or enforce final JSON output after a run.
- Plain and JSON agents are covered at the run-preparation layer; JSON content validity remains a future harness/enforcement concern.
- The rendered `<artifacts>` contract now includes an `AgentQ result mode` block derived from effective config, so frontmatter and CLI override choices are visible to the provider.
